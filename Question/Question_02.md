# Закріплення матеріалу

## Яка повинна бути перша інструкція у Dockerfile

Перша інструкція у Dockerfile – це **`FROM`**, яка визначає базовий образ для створення нового контейнера. Наприклад:

```
FROM ubuntu:latest
```

Це означає, що контейнер буде базуватися на останній версії Ubuntu.

## Які інструкції можна використовувати у Dockerfile

У Dockerfile можна використовувати такі основні інструкції:

- **`FROM`** – визначає базовий образ.
- **`RUN`** – виконує команди всередині контейнера під час його побудови.
- **`CMD`** – задає команду, яка буде виконуватися при старті контейнера.
- **`ENTRYPOINT`** – визначає основну команду для виконання в контейнері.
- **`COPY`** – копіює файли з локальної системи в контейнер.
- **`ADD`** – копіює файли та може розпаковувати архіви.
- **`ENV`** – задає змінні середовища.
- **`EXPOSE`** – відкриває порти для роботи контейнера.
- **`WORKDIR`** – задає робочу директорію.
- **`VOLUME`** – створює точку монтування для збереження даних.
- **`LABEL`** – додає мета-дані до образу.

## Що робить інструкція RUN

Інструкція **`RUN`** виконує команду під час створення образу та зберігає результат у новому шарі файлової системи контейнера. Наприклад:

```bash
RUN apt-get update && apt-get install -y nginx
```

Ця команда оновлює пакети та встановлює Nginx.

## Для чого потрібна інструкція ENTRYPOINT

Інструкція **`ENTRYPOINT`** визначає основну команду, яка буде виконуватися під час запуску контейнера. Вона дозволяє створювати більш гнучкі контейнери. Наприклад:

```bash
ENTRYPOINT ["nginx", "-g", "daemon off;"]
```

Це дозволяє контейнеру завжди запускати Nginx у фоновому режимі.

## Як опублікувати образ у Docker Hub

Щоб опублікувати образ у Docker Hub, виконайте наступні кроки:

1. Авторизуйтесь у Docker Hub:

```bash
docker login
```

2. Побудуйте образ:

```bash
docker build -t username/myimage:latest .
```

3. Завантажте образ у Docker Hub:

```bash
docker push username/myimage:latest
```
